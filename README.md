# PaperLifeOS

> 一个事件驱动、可调试的人生系统架构

---

## 为什么有这个仓库

这个系统不是从理论里长出来的，是从一次次"不对劲"里逼出来的。

我试过市面上大部分生活系统：GTD、OKR、习惯追踪、量化自我、各种 Notion 模板。它们在某些阶段有用，但都有一个共同的问题——**把人生当成一个可以预先设计并严格执行的项目**。当状态好的时候，这些系统运转流畅；但当低谷来临、当身份模糊、当情绪和身体发出信号时，这些系统要么继续运转制造出更多待办债务，要么直接崩溃留下一个更大的空洞。

更具体地说，我遇到过这些死局：

- **量化指标的暴政**：打卡中断一次就产生强烈的失败感，系统从工具变成审判者
- **待办清单的异化**：为了清空清单而做不重要的事，真正的优先级被淹没
- **正向激励的失效**：当强烈负反馈来袭时，"设定小目标""奖励自己"这类技巧完全无力
- **身体信号被忽略**：脑子说"再做一点"，但肩膀紧、呼吸浅、入睡困难——这些信号没有进入任何系统

2026 年 3-4 月，我入职摩尔线程前的 gap 月，经历了一次比较深的状态低谷。那段时间我意识到：**我需要的不是一个更完美的计划系统，而是一个能在顺境和逆境中都保持运转、能识别直觉异常、能在崩溃时进入调试模式的系统。**

PaperLifeOS 就是这个尝试。它不是教你如何"成功"，而是帮助你在真实环境中**运行、观察、调整**。

## 核心设计

PaperLifeOS v0.3 是一个三层交互的事件驱动系统：

- **实践层**：产生真实事件和可观测结果
- **叙事层**：给事件赋予方向和意义，但不垄断裁决权
- **锚定层**：通过身体信号、他人行动、时间/结果来证伪叙事

三者之间的动态边界称为**张力带**，直觉异常从这里涌现，进入**调整器**——输出一个当下可启动的具体行动、一个有意识的非行动，或进入调试模式的判断。

关键机制包括：

- **顺境锚定**：好消息时主动生成可被证伪的负面预测，防止锚定器失效
- **逆境锚定**：坏消息时寻找未被完全摧毁的锚定点，防止叙事过度崩溃
- **权力/安全评估**：在关系事件中先评估风险等级，再决定是否表达真实想法
- **最小记录原则**：只允许轻量外部化记忆，禁止待办清单、打卡表、量化指标
- **调试模式 / 半调试模式**：重大冲击或慢性压力时的系统降级运行协议

完整架构文档见 [`人生系统架构-v0.3.md`](人生系统架构-v0.3.md)，真实运行记录见 [`人生系统架构-v0.3-故事板.html`](人生系统架构-v0.3-故事板.html)。

## 如何使用这个仓库

### 作为读者

1. 从 [`人生系统架构-v0.3.md`](人生系统架构-v0.3.md) 了解完整架构
2. 从 [`人生系统架构-v0.3-故事板.html`](人生系统架构-v0.3-故事板.html) 看真实场景下的系统运行记录
3. 如果你也用过其他生活系统并感到某种"不对劲"，这里的框架可能提供一种不同的视角

### 作为实践者

这个系统的设计目标是**低维护成本、高感知输出**。日常只需要：

1. **事件发生后**，快速扫描三层：我做了什么（实践）？我怎么看这件事（叙事）？我的身体/关系/结果反馈是什么（锚定）？
2. **感知到直觉异常时**，用调整器输出一个最小行动或非行动
3. **必要时做最小记录**：`事件 → 身体信号 → 异常关键词 → 调整器输出`

不需要每天复盘，不需要追踪指标，不需要维持连续打卡。

## AI Skill：让系统主动运行

这个仓库附带一个 Claude Code / Claude Desktop 可用的 skill——**PaperLifeOS**。它的作用是让 AI 能够理解并协助操作这个系统，把"读文档"变成"直接运行"。

### Skill 能做什么

- **完整运行模式**：你描述一个真实事件，AI 按三层架构分析并输出调整器结果
- **辅助分析模式**：你只想"分析一下"某个信号或层间关系，AI 做引导式反思而不强行给行动指令
- **概念解释模式**：你问某个概念，AI 给出简洁解释并提醒你这是用来运行的

### 安装方法

#### 方法一：Claude Code（推荐）

1. 确保你的 Claude Code 版本支持 user skills（通常需要较新版本）
2. 将本仓库中的 [`PaperLifeOS-SKILL.md`](PaperLifeOS-SKILL.md) 复制到你的 skills 目录：
   ```bash
   # macOS/Linux 默认路径
   cp PaperLifeOS-SKILL.md ~/.claude/skills/PaperLifeOS/SKILL.md
   
   # 或如果你使用 .agents 路径
   cp PaperLifeOS-SKILL.md ~/.agents/skills/PaperLifeOS/SKILL.md
   ```
3. 重启 Claude Code 或重新加载 skills
4. 当你在 PaperLifeOS 仓库中工作，或对话中提到"人生系统""调整器输出""身体信号""叙事层""锚定层"等概念时，skill 会自动触发

#### 方法二：Claude Desktop（Projects）

1. 在 Claude Desktop 中创建一个 Project
2. 将本仓库的文件作为上下文添加到 Project 中
3. 在 Project Instructions 中粘贴 [`PaperLifeOS-SKILL.md`](PaperLifeOS-SKILL.md) 的内容
4. 与该 Project 对话时，AI 会按 skill 规则响应

### 使用示例

**场景 1：深夜写代码后身体发紧**
> 我在 PaperLifeOS 仓库里。今天凌晨 2 点终于把一个 bug 修好了，测试全绿，特别兴奋，然后我又打开了一个新文件想再优化一下。现在脖子很僵，眼睛干涩。

AI 会识别为深夜写代码场景，执行顺境锚定，分析身体信号为"紧"，输出强制休息的具体行动。

**场景 2：只想分析一个信号**
> 我最近总是感到肩膀紧，但不知道这意味着什么

AI 会进入辅助分析模式，将"紧"映射到锚定层，解释其常见来源（防御、压抑、未表达的真实想法），用引导式问题帮你看到层间关系。

**场景 3：问概念**
> 张力带是什么意思？

AI 会给出简洁解释，然后提醒：这个系统是用来运行的，不是用来理解的。
